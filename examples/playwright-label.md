## playwright

> Known server: publisher trust is **High**.

### Expected Capabilities

- **High:** `browser.control` — Playwright MCP gives the model browser automation capabilities such as navigation, clicking, typing, form filling, screenshots, and tab interactions.
- **Medium:** `network.fetch` — The server can access network resources through browser navigation and page interactions.
- **Critical:** `code.execution` — Playwright MCP documents an unsafe tool (browser_run_code_unsafe) that can run Playwright code in the server process. This is a well-known, Microsoft-maintained package; the tool exists for advanced use cases but should be treated as RCE-equivalent unless disabled by policy or client-side tool allowlisting.
- **Medium:** `browser.session_persistence` — The browser profile may persist cookies, local storage, and login state between sessions unless isolated mode is used.
- **Medium:** `filesystem.read_possible` — Some browser file interactions can reference local file paths. This is not the same as unrestricted filesystem access.

### Install Risks

- **Medium:** `npx-latest` — The npx package uses @latest, which can change over time.

### Recommendations

- Use an isolated browser profile and avoid sensitive logged-in sessions.
- Use --allowed-origins or --blocked-origins to constrain browser network access.
- Require explicit approval for this tool, disable it via client tool allowlists, or avoid enabling it in sensitive environments.
- Add --isolated unless persistent login state is intentionally required.
- Avoid exposing sensitive files through upload/drop tools and review any configured file access flags.
- Pin the package to a specific version, for example @playwright/mcp@<version>.
